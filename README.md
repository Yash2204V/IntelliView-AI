# Intelli View AI Platform

An AI-powered mock interview platform built with Next.js (App Router) that simulates real interview rounds with:

- Role- and company-specific questions generated by Groq
- Tech stack & experience–aware question difficulty (great for interns/freshers)
- Voice-based answering with browser speech recognition
- Live and post-interview feedback
- Data stored in MongoDB with simple session-based auth

## Features

- **Onboarding flow**
  - Pick a role (e.g. Software Engineer, Product Manager).
  - Choose a target company.
  - Enter your primary tech stack and experience level (Fresher, Junior, Mid, Senior).

- **Interview room**
  - Groq generates 5 simple, stack-focused questions.
  - Questions are spoken using browser text-to-speech.
  - Answer using your microphone; speech recognition transcribes your response.
  - Per-question AI feedback appears after each answer.

- **Evaluation & results**
  - All Q/A pairs are evaluated by Groq.
  - Overall metrics: **clarity, confidence, pace, engagement** (0–100).
  - Per-question scores for how well you answered each specific question.
  - A results dashboard with charts, question-by-question analysis, and improvement tips.

- **History**
  - Completed interviews are stored in MongoDB.
  - Dashboard shows your past interviews with scores.

## Tech Stack

- **Frontend**: Next.js App Router, React, TypeScript, Tailwind CSS, Framer Motion
- **UI components**: Custom components in `components/ui` (cards, charts, dialogs, etc.)
- **AI**: [Groq](https://groq.com/) via `groq-sdk`
- **Database**: MongoDB via official Node.js driver
- **Auth**: Cookie + session collection (`users` and `sessions` collections)

## Getting Started

### Prerequisites

- Node.js 18+ (recommended)
- pnpm, npm, or yarn (examples use **pnpm**)
- A MongoDB instance (local or cloud)
- A Groq API key

### 1. Clone the repo

```bash
git clone https://github.com/Yash2204V/IntelliView-AI.git
cd IntelliView-AI
```

### 2. Install dependencies

```bash
pnpm install
# or
npm install
```

### 3. Configure environment variables

Create a `.env.local` file in the project root with at least:

```bash
MONGODB_URI=mongodb+srv://<user>:<password>@<cluster>/<db-name>?retryWrites=true&w=majority
GROQ_API_KEY=your_groq_api_key_here
```

- `MONGODB_URI` is used by `lib/db.ts` to connect to MongoDB.
- `GROQ_API_KEY` is used by `lib/groq-client.ts` to call Groq models.

> In development, interview routes can bypass strict auth to make local testing easier, but production still uses sessions stored in MongoDB.


### 5. Run the dev server

```bash
pnpm dev
# or
npm run dev
```

Then open `http://localhost:3000` in your browser.

## Project Structure

Key directories and files:

- `app/`
  - `page.tsx` – Landing page.
  - `onboarding/page.tsx` – Role/company/tech stack onboarding before interviews.
  - `interview/[id]/page.tsx` – Interview room shell that loads one interview.
  - `results/[id]/page.tsx` – Results dashboard page.
  - `dashboard/page.tsx` – Interview history.
  - `api/`
    - `auth/` – Signup, login, logout endpoints.
    - `interviews/`
      - `create/route.ts` – Start a new interview.
      - `question/route.ts` – Get next AI-generated interview question.
      - `feedback/route.ts` – Get AI feedback for a given answer.
      - `complete/route.ts` – Finalize an interview and trigger evaluation.
      - `[id]/route.ts` – Fetch an interview by ID.
      - `[id]/results/route.ts` – Fetch results & metrics for an interview.
      - `history/route.ts` – List past interviews for the user.

- `components/`
  - `interview-room.tsx` – Main interview experience (voice input, TTS, feedback).
  - `results-dashboard.tsx` – Visualizes interview metrics & per-question analysis.
  - `voice-input.tsx` – Speech recognition wrapper.
  - `waveform-visualizer.tsx` – Microphone waveform visualization.
  - `live-feedback.tsx` – Lightweight live heuristics during the interview.

- `lib/`
  - `db.ts` – MongoDB connection & index setup.
  - `groq-client.ts` – Helpers for:
    - `generateInterviewQuestion`
    - `generateFeedback`
    - `evaluateInterview` (overall metrics)
    - `evaluatePerQuestionScores` (per-question scores)
  - `utils.ts` – Misc utilities.

## How Evaluation Works (High Level)

1. During the interview, each question and your corresponding answer are stored.
2. When the interview is marked complete (`/api/interviews/complete`):
   - `evaluateInterview` computes overall metrics (clarity, confidence, pace, engagement).
   - `evaluatePerQuestionScores` computes a 0–100 score **per question**.
3. These metrics are saved on the interview document and shown in the results dashboard.
4. The prompts explicitly:
   - Ignore speech-recognition glitches and nervous wording.
   - Focus on key concepts and technical correctness.
   - Use conservative bands (weak / partial / good / strong) instead of random scores.
